What is EC2?

Provides Infrastructure-as-a-Service (IaaS) capabilities:

  • Renting virtual machines (EC2)
  • Storing data on virtual drives (EBS)
  • Distributing load across machines (ELB)
  • Scaling services automatically (ASG)

User Data Script

We can use this script to bootstrap instances, e.g. installing updates or software

Runs with root permissions

Instance Types

AWS uses a specific naming convention for their instance types: e.g. m5.2xlarge

m - instance class

5 - generation

2xlarge - size within the class

General purpose

Diverse workloads such as web servers

Balance between compute, memory, and networking

Compute Optimised

Designed for compute intensive tasks such as :

  • Batch processing workloads
  • Media transcoding
  • High Performance Computing
  • Scientific modelling & Machine Learning

Memory Optimised

Fast performance for workloads that process large data sets in-memory

  • High performance databases
  • Distributed web-scale cache stores
  • Real-time processing of unstructured data

Storage Optimised

Designed for storage intensive tasks that need high sequential read/write performance

  • High frequency Online Transaction Processing (OLTP)
  • Relational & NoSQL DBs
  • Data warehousing

Security Groups

Control how traffic is allowed into/out of instances

Only contain allow rules

Can reference by IP or by security group

Effectively act as firewalls

Regulate:

  • Access to ports
  • Authorised IP ranges
  • Control of inbound/outbound network traffic

Can be attached to multiple instances.

It’s good practice to maintain one separate group for SSH access

By deault all outbound traffic is authorised, and all inbound traffic is blocked

Payment Types

On Demand

Pay for usage:

  • Linux/Windows - billing per second
  • Other OSs - billing per hour

Has the highest cost, but no upfront payment

No long term commitment

Recommended for short-term and un-interrupted workloads where you can’t predict how an application will behave.

Reserved Instances

Reserve a specific instance attribute (e.g. Instance Type, Region, Tenancy, OS etc)

Can be for 1 or 3 years

Payment Options: No upfront, partial upfront, all upfront

Reserved instance’s scope: regional or zonal

Recommended for steady-state usage applications (e.g. databases)

Can buy and sell in the Reserved Instance Marketplace

Convertible Reserved Instance: can change instance type, family, OS, scope, and tenancy.

Savings Plans

Discount on long-term usage

Commit to a certain type of usage (e.g. $10/hour for 1 or 3 years)

Usage beyond Savings Plan is billed at the On-Demand rate

Locked to a specific instance family and region

Flexible across: instance size, OS, tenancy (host, dedicated, default)

Spot Instances

Define a max price, and provision instances when the spot price drops below this price

Spot price is dependant on demand, and fluctuates

Most cost effective instances - substantial discount (e.g. 90% compared to On-Demand)

Instances will be lost if the spot price increases above your maximum price

Useful for interruptible workloads: batch jobs, data analysis etc

Capacity Reservations

Reserve On-Demand instance capacity in a specific AZ

Guarantees you have capacity when needed

No time commitment = no billing discounts

Combine with Regional Reserved Instances and Savings Plans to benefit from discounts

Charged at On-Demand rate whether or not you use it

Suitable for short-term, uninterrupted workloads that need to be in a specific AZ

Tenancy Types

Dedicated Hosts

Physical server with EC2 capacity fully dedicated to you

Address compliance requirements and use existing server-bound software licenses

Most expensive option for EC2 capacity

Useful for when you have complicated licensing models, or strong compliance needs

Pricing: On-Demand, and Reserved (1 or 3 years)

Dedicated Instances

Instances run on hardware that’s dedicated to you

May share hardware with other instances in the same account

No control over instance placement (can move hardware after instance stop)
